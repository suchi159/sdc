# Render Deployment Guide — SDC Certifications

This guide walks you through deploying the SDC Certifications platform to [Render.com](https://render.com) from scratch.

---

## Prerequisites

| Requirement | Notes |
|---|---|
| Render account | Free tier works for testing; use **Starter** or higher for production |
| MySQL database | PlanetScale (free tier), TiDB Cloud (free tier), or Render's own MySQL add-on |
| AWS S3 bucket | For file storage (certificates, avatars, bulk imports). Cloudflare R2 and Backblaze B2 are free-tier alternatives |
| Resend account | For transactional emails — [resend.com](https://resend.com) (free tier: 3,000 emails/month) |
| Stripe account | For voucher purchases — test keys work immediately, live keys after KYC |
| OpenAI API key | For AI-powered exam generation and psychometric analysis |

---

## Step 1 — Connect your GitHub repository

1. Push this repository to GitHub (or fork it).
2. In the Render dashboard, click **New → Web Service**.
3. Connect your GitHub account and select the repository.

---

## Step 2 — Configure the Web Service

| Setting | Value |
|---|---|
| **Runtime** | Node |
| **Region** | Singapore (or closest to your users) |
| **Branch** | `main` |
| **Build Command** | `pnpm install && pnpm build` |
| **Start Command** | `node dist/index.js` |
| **Health Check Path** | `/api/health` |
| **Plan** | Starter ($7/mo) recommended — free tier spins down after inactivity |

---

## Step 3 — Set Environment Variables

In **Dashboard → Your Service → Environment**, add the following variables:

### Required

| Variable | Description | Example |
|---|---|---|
| `NODE_ENV` | Must be `production` | `production` |
| `DATABASE_URL` | MySQL connection string | `mysql://user:pass@host:3306/db` |
| `JWT_SECRET` | 32+ char random secret for session cookies | `openssl rand -base64 32` |
| `APP_URL` | Your Render service URL (no trailing slash) | `https://sdc-certifications.onrender.com` |

### LLM (AI features)

| Variable | Description | Default |
|---|---|---|
| `OPENAI_API_KEY` | OpenAI or compatible API key | — |
| `OPENAI_API_URL` | API base URL | `https://api.openai.com` |
| `OPENAI_MODEL` | Model name | `gpt-4o-mini` |

**Free/cheap alternatives to OpenAI:**
- [Together AI](https://together.ai) — `OPENAI_API_URL=https://api.together.xyz`, model: `mistralai/Mixtral-8x7B-Instruct-v0.1`
- [Groq](https://groq.com) — `OPENAI_API_URL=https://api.groq.com/openai`, model: `llama3-8b-8192`

### File Storage (AWS S3 or compatible)

| Variable | Description |
|---|---|
| `AWS_ACCESS_KEY_ID` | S3 access key |
| `AWS_SECRET_ACCESS_KEY` | S3 secret key |
| `AWS_REGION` | Bucket region (e.g. `us-east-1`) |
| `AWS_S3_BUCKET` | Bucket name |
| `AWS_S3_ENDPOINT` | *(Optional)* Override for R2/B2/MinIO |

**Free S3-compatible alternatives:**
- [Cloudflare R2](https://developers.cloudflare.com/r2/) — 10 GB free, set `AWS_S3_ENDPOINT=https://<account>.r2.cloudflarestorage.com`
- [Backblaze B2](https://www.backblaze.com/b2/cloud-storage.html) — 10 GB free

### Stripe

| Variable | Description |
|---|---|
| `STRIPE_SECRET_KEY` | From Stripe Dashboard → API Keys |
| `STRIPE_WEBHOOK_SECRET` | From Stripe Dashboard → Webhooks |
| `VITE_STRIPE_PUBLISHABLE_KEY` | Publishable key (baked into frontend at build time) |

> **Important:** `VITE_*` variables are embedded into the frontend bundle during `pnpm build`. If you change them, you must trigger a new deploy.

### Email

| Variable | Description |
|---|---|
| `RESEND_API_KEY` | From [resend.com/api-keys](https://resend.com/api-keys) |
| `RESEND_FROM_EMAIL` | Verified sender address (e.g. `noreply@yourdomain.com`) |

### Frontend (build-time)

| Variable | Description |
|---|---|
| `VITE_APP_TITLE` | Browser tab title |
| `VITE_APP_LOGO` | *(Optional)* URL to your logo image |

## Step 4 — Set up the Database

Run the database migrations after your first deploy:

```bash
# Option A: Using Render Shell (Dashboard → Shell tab)
pnpm drizzle-kit generate
pnpm drizzle-kit migrate

# Option B: Run locally against your production DATABASE_URL
DATABASE_URL=mysql://... pnpm drizzle-kit migrate
```

---

## Step 5 — Configure Stripe Webhook

1. In [Stripe Dashboard → Webhooks](https://dashboard.stripe.com/webhooks), click **Add endpoint**.
2. Set the URL to: `https://your-render-url.onrender.com/api/stripe/webhook`
3. Select events: `checkout.session.completed`, `payment_intent.succeeded`, `invoice.paid`
4. Copy the **Signing secret** and set it as `STRIPE_WEBHOOK_SECRET` in Render.

---

## Step 6 — Deploy

Click **Manual Deploy → Deploy latest commit** in the Render dashboard, or push to `main` to trigger auto-deploy.

The health check at `/api/health` will confirm the service is running.

---

## Step 7 — Create the First Admin User

After the database is migrated:

1. Register a new account at `https://your-render-url.onrender.com/register`
2. In your database, update the user's role to `admin`:

```sql
UPDATE users SET role = 'admin' WHERE email = 'your@email.com';
```

## Troubleshooting

| Issue | Solution |
|---|---|
| Build fails with `pnpm: command not found` | Render auto-installs pnpm; ensure `package.json` has `"packageManager": "pnpm@..."` |
| `DATABASE_URL` connection refused | Check SSL: add `?ssl={"rejectUnauthorized":true}` for PlanetScale/TiDB |
| Stripe webhook returns 400 | Ensure `STRIPE_WEBHOOK_SECRET` matches the Render endpoint's signing secret |
| Emails not sending | Verify your sender domain in the Resend dashboard |
| AI features return 500 | Check `OPENAI_API_KEY` is set and the model name is correct for your provider |
| Files not uploading | Verify S3 bucket exists, credentials are correct, and the bucket has public read ACL (or use presigned URLs) |

## render.yaml (Infrastructure as Code)

A `render.yaml` file is included in the repository root. You can use it with Render's [Blueprint](https://render.com/docs/infrastructure-as-code) feature for one-click deploys:

1. In the Render dashboard, click **New → Blueprint**.
2. Connect your repository — Render will detect `render.yaml` automatically.
3. Fill in the `sync: false` variables (DATABASE_URL, API keys, etc.) when prompted.

## Architecture on Render

```
Internet
    │
    ▼
Render Web Service (Node.js)
    ├── Express server (port from $PORT)
    ├── tRPC API  (/api/trpc)
    ├── REST API  (/api/v1)
    ├── Stripe webhook  (/api/stripe/webhook)
    └── Static frontend (dist/public/)
         └── React SPA (Vite build)

External services:
    ├── MySQL / PlanetScale / TiDB  ← DATABASE_URL
    ├── AWS S3 / R2 / B2            ← AWS_* vars
    ├── OpenAI / Together / Groq    ← OPENAI_* vars
    ├── Resend                      ← RESEND_* vars
    └── Stripe                      ← STRIPE_* vars
```
